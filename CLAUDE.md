## Architecture

Two strictly separated layers (plus a thin CLI over the first):

1. **`swift/Engine/` ‚Äî one SwiftPM package, TWO library modules.** `Sources/Anzan/` is the LANGUAGE (ÊöóÁÆó, mental abacus calculation ‚Äî use the name Anzan in user-facing copy): `Lexer/` ‚Üí `Parser/` (Pratt) ‚Üí `Eval/` + `Number/` + `Functions/`, fronted by the `Calculator` facade (`evaluate(String) -> Result<EvalOutcome, EngineError>` ‚Äî `.value` updates `ans`; `.functionDefined`/`.dataDefined`/`.documentation` don't), which owns the `EvaluationEnvironment`; Documentation/EngineError sit beside it. Anzan knows NOTHING about grids or files ‚Äî hosts wire cells in via the Calculator's resolver closures, and the module boundary now enforces that (don't add a Sheet/Persistence import to Anzan; `Calculator.restoreSession(from: Workbook)` lives in SorobanEngine's Calculator+Workbook.swift for exactly this reason). `Sources/SorobanEngine/` is the HOSTING layer ‚Äî `Sheet/` (Spreadsheet, SheetStore, Cell, CellAddress, CellFormat, NamedCells, Controls) + `Persistence/` (Workbook codec, journal, package, DataStore) ‚Äî and `@_exported import Anzan`s, so depending on SorobanEngine gives the whole engine (the app never imports Anzan directly). Cross-module internals use the `package` access level (Parser.parse, Lexer.tokenize, FunctionRegistry, CivilDate, UserFunction's init); tests `@testable import` BOTH modules.
2. **`swift/App/` ‚Äî SwiftUI macOS app** (target defined in `swift/project.yml`, bundle id `com.alleato.Soroban`). `CalculatorSession` (@Observable, @MainActor) wraps `Calculator` and owns the log entries, persisted ‚Üë/‚Üì input history, the `SheetModel` (grid state/persistence), the `WorkbookManager` (Save/Open document control), and `activeView` (log ‚Üî grid, ‚åò\; in grid mode the input bar hides and the toggle floats bottom-right). `SheetModel` is ONE type across `Session/SheetModel/` ‚Äî @Observable requires every STORED property in the class body (all in SheetModel.swift, grouped by owner), while behavior lives in per-concern `SheetModel+*.swift` extensions; cross-file members are internal by necessity, so treat anything commented as extension state as private to its section. The grid views split the same way: SpreadsheetView (container/keyboard/resize) ‚Üí GridRowView ‚Üí CellView.swift (+ PaletteColor mapping) ‚Üí CellControls.swift / CellEditorView.swift. Views read every color/font from `ThemeManager.current` ‚Äî never hardcode colors.
3. **`swift/Engine/Sources/SorobanCLI` ‚Äî the `soroban` executable** (a product of the engine package; build/install commands above). Depends on **Anzan only** ‚Äî the CLI is the language without the app, and deliberately has no sheet layer. One `Calculator` per invocation; mode by shape: arguments ‚Üí one-shot shared session, piped stdin ‚Üí line-per-line (plain output, exit 1 if any line failed), tty ‚Üí linenoise REPL (tab completion via `completions(forPrefix:)` + `trailingIdentifier(of:)`, gray `name(` signature hints from `FunctionDoc`, ‚Üë/‚Üì history persisted to `~/.soroban_history`). Errors render the same column-accurate caret as the app. **Keep it plumbing-thin** ‚Äî any behavior worth testing belongs in the engine (the target is excluded from the coverage report for exactly that reason). The `linenoise-swift` dependency is declared CLI-target-only and pinned to a COMMIT (its newest tag predates Swift 5); the library target must stay BigInt-only.

### BigDecimal (the core invariant)

`BigDecimal` = `BigInt` significand √ó 10^exponent, always normalized (no trailing zeros). `+ ‚àí √ó`, integer `^`, and `%` are **exact**; `/`, `sqrt` round to `PrecisionContext.current` significant digits (default 50, banker's rounding). Transcendentals (`exp`, `ln`, `log`, trig, non-integer `pow`) round-trip through `Double` (~15 digits) ‚Äî that fallback is deliberately confined to `BigDecimal.viaDouble(...)` in `BigDecimal+Math.swift`; route any new inexact function through that seam so a future arbitrary-precision upgrade happens in one place. Don't introduce `Double` math anywhere else in the engine.

### Value (the engine's value type)

Every expression evaluates to a `Value` (Eval/Value.swift): `.number(BigDecimal)` / `.string` / `.array([Value])` / `.map([MapEntry])` / `.record(RecordValue)` (see Data types) / `.fixedInt(FixedInt)` (see Fixed-width integers) / `.fixedDecimal(FixedDecimal)` (see Fixed-precision decimals) / `.host(any HostObject)` (see Workbook reflection). Structures nest freely, are **immutable** (no element assignment ‚Äî rebind the variable), and render canonically (`description` re-parses to an equal value ‚Äî that's how structured variables persist in workbooks via the same string mechanism numbers use; `Workbook.parsedVariables` folds literals back through `Value.init(parsing:)`; records can't fold ‚Äî they restore by evaluation, see Workbooks). Maps keep insertion order; `==` is deep and order-insensitive for maps; ordering comparisons require numbers. `+` concatenates when either side is a string; every other operator is numeric (`asNumber(for:)` gives typed errors). Truthiness (if-conditions, ‚àë bounds/terms) requires numbers. `true`/`false` are constants = 1/0 (in `ReservedNames` and the Environment subscript). **Cell resolvers still speak BigDecimal** ‚Äî cells are scalar; the evaluator wraps. Indexing is **0-based** (user decision; `arr[0]`, also strings); map access is `m.key` / `m["key"]` (keys case-sensitive, like variables).

**Evaluator frame-size note**: the big `switch` keeps heavyweight cases (call arguments, reductions, literals) extracted into private methods ‚Äî debug builds give the switch ONE frame holding every case's locals. Stack segmentation removed the SIGBUS risk, but slim frames still mean fewer segment hops (each hop spawns a thread), so keep new fat cases extracted.

### Language modes (presentational dialects)

`LanguageMode` (`Anzan/LanguageMode.swift`: `.normal`/`.programmer`/`.finance`) is an input/display DIALECT, not a semantic switch ‚Äî the canonical AST, evaluation, storage, and codec are mode-blind; only which glyphs you type and read changes. Full spec: `docs/MODES.md`. Invariants: the **parser is mode-parameterized** (`Parser.parse(_, mode:)`) ‚Äî in `.programmer`, `^`‚Üí`bitXor`, `&`‚Üí`bitAnd`, `|`‚Üí`bitOr`, `<<`/`>>`‚Üí`bitShift` (`>>` negates the count), `%`‚Üí`mod`, prefix `~`‚Üí`bitNot`, with **Python precedence** (a bitwise band `| ^ & << >>` inserted between comparison and additive, every loop gated on `mode == .programmer`; power becomes `pow()`); out-of-mode glyphs raise `modeOperatorError`. The **renderer is mode-parameterized** too (`Expression.sourceText(mode:)`; `var sourceText` is the `.normal` delegate) ‚Äî it re-spells the canonical bitwise/mod calls and the power/percent nodes as glyphs in `.programmer`, function-fallback otherwise. **Canonical (`.normal`) is the only thing stored/transported**, so reload/replay are mode-free AND `.normal` must stay byte-identical to the pre-modes grammar (the regression oracle ‚Äî keep it green at every change). Scope is **log-only**: `Calculator.mode` drives `evaluate` (the log); `evaluateFormula` (cells) always parses `.normal`. Hosts: CLI `:mode` (`SorobanCLI/main.swift`, REPL + pipe); app `CalculatorSession.mode` (UserDefaults-persisted like the theme, Settings picker + input-bar affordance, drives `calculator.mode`). New tokens `<< >> & | ~` lex mode-agnostically (Token.swift/Lexer.swift); the parser gates them. Existing log entries display verbatim (inert records) ‚Äî the spec's uniform historical re-skin is deferred.

### Fixed-width integers (`Int`/`UInt`)

`FixedInt` (`Anzan/Eval/FixedInt.swift`: `value: BigInt` √ó `bits` √ó `signed`) is the payload of `Value.fixedInt`, built by the parameterized `Int(value, bits)` / `UInt(value, bits)` builtins AND the per-width `Int8‚Ä¶Int256` / `UInt8‚Ä¶UInt256` constructors (ProgrammerFunctions.swift; widths 8/16/32/64/128/256, signed + unsigned ‚Äî the per-width forms are generated from `FixedInt.allowedWidths`, one place for the width set, and are the **canonical** spelling). Full spec: `docs/FIXED-WIDTH.md`. **Checked, not modular** ‚Äî `FixedInt.init` range-validates, and arithmetic overflow/underflow THROWS, never wraps (the exact ethos at the bounds). Arithmetic is intercepted in `Evaluator.apply` (one `if FixedInt.isInvolved` before `asNumber`, so the numeric/decimal path is untouched) ‚Üí `FixedInt.applyBinary`: the **mixing matrix** ‚Äî largest width wins; sign never promotes (mismatch ‚Üí error); `Decimal`‚Üîfixed ‚Üí error; a plain integer *number* adopts the type, range-checked (fractional ‚Üí error); count slots (the `^` exponent, shift amount) are exempt. Bitwise (`bitAnd/Or/Xor` reduce, `bitShift`, `bitNot`/`~`) is type-preserving via **two's-complement over the width** (`FixedInt.applyBitwise`/`bitwiseNot`; `pattern`/`decode` encode signed values; the bit builtins moved to `applyValues:` so they SEE fixedInts, plain non-negative path unchanged). Outside typed arithmetic a fixedInt coerces to its `.number` (in `asNumber`/`flattenedNumbers`), so comparison/truthiness/`sum`/cell-display treat it as its number; `==` is numeric (`Int8(5) == 5`). Persists like a record ‚Äî `description` is the per-width form (`Int32(255)`), restored by re-evaluation. Adding the `.fixedInt` case touched every exhaustive `Value` switch (Value, Evaluator, JSON, DataFunctions, Spreadsheet display) ‚Äî the compiler finds them. Deferred: typed-param dispatch (`f(x: Int32) = ‚Ä¶`); radix display (always shows the canonical per-width form).

### Binary bit-editor + ans-prefix (Programmer-mode input)

Two pure, engine-tested seams (`BinaryViewTests`/`AnsPrefixTests`) with thin app glue. `BinaryView` (`Anzan/Eval/BinaryView.swift`) is the model behind the app's macOS-Calculator-style bit grid: `BinaryView.make(for: Value, preferredWidth:)` ‚Üí `Result<BinaryView, Unavailable>` ‚Äî a `fixedInt` edits at its own width/sign (two's-complement); a plain NON-negative integer edits as an unsigned register at a chosen width (8/16/32/64/128, auto-bumped to fit); `.notAnInteger` / `.negative` (use a signed `Int‚Ä¶`) / `.tooWide` (>256 bits) disable it. `flippingBit(_:)` returns a new view; `.value` reconstructs the original kind. `Calculator.ansPrefixed(_, mode:)` (Calculator.swift) is the SpeedCrunch continuation: a leading binary operator on an otherwise-empty line ‚Üí `ans`-prefixed (`%`/`& | << >>` lead only in `.programmer`; `-` included). App side (`CalculatorSession` binary* members + `BinaryEditorView.swift`): the overlay is Programmer-mode-only (gated in ContentView, default-on first run then your choice sticks, ‚å•‚åòB + a ‚úï button + the View menu hide/show it), flips stage a `binaryDraft` (live, no log spam, reset on any submit), `useBinaryValue()` INSERTS the value into the input line (a `0b‚Ä¶` literal for a plain integer, the typed constructor for an `Int‚Ä¶`) rather than posting to the log ‚Äî you fold it into an expression and submit when ready; `applyAnsPrefixIfNeeded(old:new:)` runs from the input bar's onChange (only on genuine typing from empty, never a programmatic recall). **Bit-field formats**: `BinaryView.FieldSpec`/`Field` + `fields(layout:)`/`setting(field:to:layout:)`/`layout(from:)`/`formatMap(_:)`/`flagFormatMap(_:)`/`numericFormatMap(_:)`/`formatValue(_:)` (engine, tested) decode/encode named bit ranges (low-packed, listed high‚Üílow). A field is **numeric**, **flags** (per-bit names, `r-x`), **enum** (an unsigned value indexing a `values` label list), **reserved** (locked must-be-zero gap), or **unused** (don't-care, editable gap); `Field.label`/`enumString`/`flagString`/`valueText` decode it, `Field.reserved`/`.unused` flag the gaps. A numeric field also carries a display **base** (2/8/10/16; nil=decimal via `normalizedBase`) ‚Äî `valueText` spells `0x‚Ä¶`/`0o‚Ä¶`/`0b‚Ä¶`, the editor parses input in that base (an explicit `0x`/`0o`/`0b` prefix wins); base is presentation-only like `color`. The loose map gains a `{bits, base}` numeric shape (`numericFormatMap`), a `{bits, values}` enum shape, and `{bits, reserved}`/`{bits, unused}` gap shapes; `formatValue(_:)` is the general encoder that mixes all of them in one layout (the floating-point/color/networking/systems presets ‚Äî e.g. DNS header flags, x86 EFLAGS with its repeated `reserved`/`flags` names ‚Äî round-trip through it). The MAC/IPv6/RGBA presets are hex. Two input shapes both read by `layout(from:)`: a loose **map** `{owner: 3}`/`{owner: ["r","w","x"]}`/`{Opcode: {bits, values}}` (presets) AND the typed **`Bits::BitFormat` record** (the docs/MODULES.md module ‚Äî `data BitField { name, bits, kind, flags, values }`). App: `CalculatorSession.activeFormat`/`activeLayout`/`binaryFields`/`setBinaryField` + built-in `binaryFormatPresets`; **saving emits the typed form** (`persistFormat`/`saveFormat`/`saveBuiltFormat` ‚Üí `ensureBitsSchema` defines the `Bits` namespace once, then `name = Bits::BitFormat(...)`), so a custom format persists as a typed workbook variable (`savedFormats` reads any record/map `layout(from:)` accepts). **Visual builder** (`BinaryEditorView` build mode, Format ‚ñæ ‚Üí Build‚Ä¶): click the open bits to claim a contiguous group, detail it (name / kind picker ‚Äî numeric/flags/enum/reserved/unused ‚Äî / flag or enum labels), Add; a live strip shows committed bands + open cells; Apply (transient, via `applyBuiltFormat` ‚Üí off-log `evaluateFormula`) or Save (named). The overlay resolves `binaryView` ONCE and uses an `Equatable` `BitButton` so a flip re-renders only the changed bit; enum fields render as a labeled `Picker`. The app target depends on **BigInt** (project.yml ‚Äî field values are BigInt).

### Fixed-precision decimals (`Decimal`)

`FixedDecimal` (`Anzan/Eval/FixedDecimal.swift`: `value: BigDecimal` rounded-to-scale + in range, `precision`, `scale`, `rounding: DecimalRounding`) is the payload of `Value.fixedDecimal`, built by `Decimal(value, precision, scale[, Rounding.X])` (AccountingFunctions; SQL `DECIMAL(p,s)` ‚Äî the money type), with short forms `Decimal(value)` (scale from the value, precision = `maxPrecision` 1000) and `Decimal(value, scale)` (that scale, precision = max); precision is capped at 1000 (matches Postgres declared NUMERIC) and the canonical `description` HIDES a max-default precision (renders `Decimal(0.5)` / `Decimal(0.50, 2)`, still re-parseable). Full spec: `docs/DECIMAL.md`. **Checked + controlled rounding**: construction rounds to `scale` (banker's default, or half-up) and **overflow-errors** if it needs more than `precision` digits; the rounding mode is **carried** and governs every later rounding. Arithmetic is intercepted in `Evaluator.apply` right after the `FixedInt` hook ‚Üí `FixedDecimal.applyBinary` (`+ ‚àí √ó √∑`): scale=max, precision=max, **rounding must match (else error ‚Äî the analogue of `Int`'s sign rule)**; a plain `Number` is **absorbed and rounded to the decimal's scale** (the money model ‚Äî *unlike* `Int`, which errors on decimal-mix); `Decimal`‚äï`Int` errors (cross bounded-family); result range-checked ‚Üí overflow (sums can overflow a tight precision ‚Äî size it like an int width); `^`/modulo error (deferred). Coerces to `.number` outside arithmetic (asNumber/`==`/cell display). `Rounding` is a reserved constant map (`Constants.rounding`, like `Json`). Half-up adds `BigDecimal.roundedHalfUp(toPlaces:)`; banker's reuses `rounded(toPlaces:)`. `description` pads to scale (`Decimal(10.50, 5, 2)`), restored by re-evaluation; adding the case touched the same exhaustive `Value` switches as `.fixedInt`.

### Adding a built-in function

**Language vs. library (the gate, specified in docs/ANZAN.md "Design rules")**: a construct may become SYNTAX/a special form only if it (1) needs unevaluated arguments, (2) binds names, or (3) is a literal shape / session mutation ‚Äî otherwise it MUST be a registry function: pure (no environment access ‚Äî the higher-order `applier` is the one sanctioned hole, passed only by the evaluator; no I/O or randomness, recalc must be reproducible) and composing with existing value types (function values, arrays) instead of inventing parallel conventions. Prefer a VALUE to a construct (`Json.Pretty` is a constant map, not enum syntax; `list(‚Ä¶)` bridges ranges to HOFs, not a new expression form). THIRD tier: a registry slot requires engine internals/algorithmic depth, a convention worth pinning, or spreadsheet arrival-vocabulary ‚Äî mere compositions (`toHex(n) = toBase(n, 16)`) stay USER-defined, because a builtin name is confiscated from user space forever; when in doubt, don't register it. SELF-HOSTING the derived builtins (implementing avg etc. as Anzan source) was considered and refused ‚Äî user functions can't be variadic, and an AST walk per call would tax every recalc; instead the "Derived builtins equal their definitions" outline in library.feature pins each bespoke implementation to its composition, exactly.

Add a `BuiltinFunction` entry to the appropriate list in `Engine/Sources/SorobanEngine/Functions/` (core/logic/trig/finance/dates/accounting/stats/data/programmer/controls ‚Äî all merged into `FunctionRegistry.standard`). **Documentation is part of registration**: `category`, `signature`, `summary`, and `examples` are required init parameters, and `DocumentationTests` evaluates every example against a seeded sheet ‚Äî an undocumented function won't compile, a broken example fails CI, and the reference window (‚åò/) picks new functions up automatically. Special forms/operators/constants need a manual `FunctionDoc` entry in `Documentation.swift`. Names are matched case-insensitively and must be unique across all lists (asserted at startup). Variadic arity is `n...Int.max`. Variables, by contrast, are case-sensitive; `ans`/`pi`/`tau`/`e`/`true`/`false` are reserved (parser rejects assignment ‚Äî see `ReservedNames` in Parser.swift).

Three implementation kinds (`BuiltinFunction.Implementation`): the `apply:` init is **numeric** ‚Äî array arguments flatten recursively in place (the structure analogue of range expansion: `sum(arr)` ‚â° `sum(A:1..A:9)`), and **arity is checked after flattening** (so `max(arr)` passes a 2+-arity check; strings/maps/functions in numeric args are typed errors). The `applyValues:` init is **structure-aware** (len/first/last/keys/values/concat in DataFunctions.swift) ‚Äî it sees raw `[Value]` and checks arity on the raw count. The `applyHigherOrder:` init additionally receives an **applier** callback (map/filter/reduce) ‚Äî the evaluator passes it when calling (`registry.call(name:arguments:applier:)`), since only the evaluator owns the environment and depth budget. User functions always get raw values (an array binds as ONE parameter).

### Function values & lambdas

`Value.function(FunctionValue)`: a bare name in expression position falls back to a function value AFTER variable lookup fails (locals ‚Üí env vars ‚Üí user functions ‚Üí builtins ‚Üí unknownVariable) ‚Äî note `rate` alone is now the finance builtin, not an unknown-variable error, so tests probing "variable is gone" must probe numerically (`100 * rate`). Lambdas `x -> body` / `(a, b) -> body` parse via speculative lookahead at the TOP of `comparison()` (every expression entry point), rewinding when no `->` follows; `->` lexes in the two-char pass before minus. Lambdas **capture the `locals` dict by value** at creation (closure; parameters shadow captures, captures shadow globals). Calls resolve registry ‚Üí user functions ‚Üí variable/local holding a `.function` value. Named references (`.builtin`/`.user(name:)`) stay symbolic and re-resolve at call time ‚Äî `alias = h` follows later redefinitions of `h`; lambdas carry their AST + captures. Rendering/persistence: `FunctionValue.description` uses `Expression.sourceText` (Parser/Expression+Source.swift ‚Äî conservative parens, contract is RE-PARSEABILITY not prettiness; extend it when adding AST cases). Persisted lambdas reload capture-free; persisted user-function references are dropped (`Value.literal` can only fold builtin references). Cells reject function-valued results like arrays/maps.

### User-defined functions

`f(x) = body` parses via speculative lookahead in `Parser.functionDefinition()` (rewinds when the shape isn't exactly `ident(idents‚Ä¶) =`, so calls stay calls). `UserFunction` (Environment.swift) stores the body **AST** plus the original `source` line for workbook serialization. Resolution is built-ins first, then user functions ‚Äî no overlap is possible because the evaluator rejects definitions whose name exists in `FunctionRegistry.standard`. Parameters bind via the `locals` dict threaded through `Evaluator.evaluate` (they shadow globals; assignments inside bodies still write globals). Recursion: TAIL CALLS are optimized ‚Äî `apply(user:)` loops at constant stack when the recursive call is the whole result of the taken if() branch (`tailStep` walks conditionals; resolution order mirrors `call(name:)` exactly ‚Äî keep them in sync), so accumulator-style recursion runs to ~1M iterations (`maxTailIterations`, the freeze guard for base-case-less tail typos). NON-tail recursion is bounded by MEMORY: when the thread's stack runs low (`nearStackLimit`, pthread introspection), evaluation continues on a fresh 16 MB thread segment (`continueOnFreshStack` ‚Äî the caller blocks, single-threaded discipline holds); `maxCallDepth` (10,000) is the runaway sanity cap. Both caps' error hints at the missing base case. `evaluateFormula` rejects definitions and assignments ‚Äî but a PLAIN `f(x) = ‚Ä¶` / `x = ‚Ä¶` typed into a cell never reaches it: it classifies as a sheet-scoped Œª/ùëñ definition (see that section). On workbook open, functions re-enter by re-evaluating their `source` lines (which carry their `# doc comments`).

### Comments, doc comments, man()

`#` comments run to end of line (lexer returns nil at `#`; a `#` inside a string stays literal). **Comments are first-class** (`Lexer.splitComment` is the string-aware splitter both hosts and the calculator use): a comment-only line is `EvalOutcome.comment(String)` ‚Äî a recorded NOTE, never a parse error, never touches `ans` (the log renders it dim via `HistoryEntry.Outcome.comment`; the CLI echoes it, exit 0); a `#‚Ä¶`-only CELL is `Cell.Content.note` ‚Üí `CellDisplay.note`, rendered dim, holding no value (skipped in ranges, errors on direct numeric reference, like text). A trailing comment on a calculation is kept: `Calculator.trailingComment(in:)` surfaces it, shown dimmed beside the result (log `HistoryEntry.note`, CLI pretty mode) and retained in a grid formula cell's raw. `standaloneComment(in:)`/`trailingComment(in:)` are the public host seams. A user function's documentation IS the trailing `# comment` of its definition ‚Äî `UserFunction.documentation` is computed from `source`, so docs persist in workbooks for free and update on redefinition; don't add a separate doc field/storage. `man name` / `manual name` / `help name` are **space-form** special forms (unix-style, NO parentheses ‚Äî `man(name)` errors with a migration hint; the argument is a NAME, never evaluated): parser ‚Üí `.helpRequest`, intercepted in `Calculator.evaluate` ‚Üí `EvalOutcome.documentation(FunctionDoc)` via `documentation(for:)`; rejected in cells; never touches `ans`; `man`/`manual`/`help` are in `ReservedNames` and `.helpRequest` renders as `man name`. The log renders it via `HistoryEntry.Outcome.info` (plain block, no "= " prefix).

### Reductions (‚àë / ‚àè)

Each symbol has two syntaxes, split by shape (mirrors math notation; user decision): a plain call is the variadic registry function ‚Äî `‚àë(‚Ä¶)` rewrites to `.call("sum", ‚Ä¶)`, `‚àè(‚Ä¶)` resolves to the `product` builtin; the **indexed forms** `‚àë_i=1^10(i^2)` / `‚àè_i=1^5(i)` (typeable: `sigma_i=‚Ä¶` / `product_i=‚Ä¶`) are **special forms** parsed by `Parser.mathReduction` into `.reduction(operation:index:lower:upper:body:)` ‚Äî the term must not be evaluated eagerly. Key parsing rules: the lexer maps `‚àë`/`‚àè` (not letters) to `identifier("sigma")`/`identifier("product")`, so `_i` follows as its own token, while typed `sigma_i`/`product_i` arrive as one identifier ‚Äî all funnel into `mathReduction`. **Bounds are signed `boundPrimary`s** (number/variable/cell/parenthesized; optional minus): NOT `unary()` (power() would eat the `^` separator) and NOT `primary()` (`n(` would parse as a call and swallow the term's parens) ‚Äî compound bounds need parens, e.g. `‚àë_i=(n-1)^10(i)`. The `sigma_`/`product_` name prefixes are reserved (`Parser.isReductionName`); `sigma` is in `ReservedNames` (`product` isn't ‚Äî it's a registry function, protected like any builtin). Evaluation binds the index via the `locals` mechanism (shadows globals, nests, composes with user functions); empty range = additive/multiplicative identity (‚àë‚Üí0, ‚àè‚Üí1); spans cap at 100,000 terms. Autocomplete lists `sigma` via a special case in `Calculator.completions` (`product` comes from the registry).

### Conditionals, ranges, dates

- `if(cond, then, else)` is a **special form** (`.conditional`) ‚Äî branches must stay lazy (the untaken one may divide by zero or recurse). Comparisons are a single non-chaining precedence level between assignment and additive, returning 1/0; all expression entry points (arguments, parens, bodies) go through `comparison()`, not `additive()`. recursion uses STACK SEGMENTATION (see User-defined functions) ‚Äî the old fixed depth (40, sized for Swift Testing's ~512 KB cooperative stacks) strangled honest fib(); now test threads and the CLI's 8 MB main thread behave identically.
- **Dependency-graph recalc**: evaluation records read edges in `ResolutionContext` (per-cell `dependents` + per-sheet `rangeDependents` rects; `definedValue` records the definition CELL as read by its consumer ‚Äî that edge is what keeps controls fast, see Sheet-scoped definitions); `setCell` invalidates only the transitive reader closure across sheets (`context.invalidate`), while LOG variable/function changes, renames, removes, and loads call `invalidateEverything`. Edges may be stale (safe: over-invalidation only). Don't revert `setCell` to whole-memo clearing ‚Äî cross-sheet dependents went stale that way.
- **Scratch persistence is snapshot + journal** (WAL): cell edits append one JSON line (`WorkbookJournal`, engine-tested) to `scratch-journal.jsonl` ‚Äî O(1) per edit, written immediately; structural changes and journal growth (256 entries) compact: full snapshot then truncate, in that order ‚Äî replay is idempotent so a crash in between is safe. All file work runs on ONE ordered chain (`enqueuePersist`: each detached task awaits the previous; closures are @Sendable hence nonisolated ‚Äî plain GCD closures formed in MainActor methods inherit isolation and SIGTRAP off-queue). Load = snapshot + journal replay. `.soroban` interchange files never contain a journal. Quit paths call `flushScratchNow()` (bounded semaphore wait).
- `.cellRange` (`A:1..B:9`) is legal **only as a function argument** ‚Äî the evaluator expands it in place via `Calculator.rangeResolver` ‚Üí `Spreadsheet.numericValues` (corners normalize; empty/text cells skipped Excel-style; error cells propagate). The `..` token is scanned in the two-char operator pass *before* the number scanner. Tests that wire resolvers must keep the `Spreadsheet` alive ‚Äî the closures capture it weakly.
- Dates are integer day serials (days since 1970-01-01) via the pure `CivilDate` helpers (Hinnant algorithms ‚Äî no Foundation Calendar except `today()`). `edate` clamps to month end. `xnpv`/`xirr` split their trailing arguments evenly into dates/flows (callers pass two equal ranges); xirr reuses finance's `solveRate`.

### Structure syntax (lexer/parser wrinkles)

- **Programmer literals**: `0xFF`/`0b1010` lex in `radixLiteral` (checked at the TOP of `number(from:)` ‚Äî `0` + x/X/b/B). Any width via BigInt; a dangling letter/digit/`.` is a LEX error, deliberately ‚Äî before this, `0xFF` parsed as `0 * xFF`, and silence was the failure mode. Bit operators (`<< & |`) were CONSIDERED AND REFUSED: `^` is power (a C-style XOR `^` would compute wrong numbers silently), `&` is reserved prospectively for Excel-style string concatenation, and the design-rules gate says bit ops are functions.

- New tokens: `[ ] { }`, `.colon`, `.dot`, `.string`. String literals are `"‚Ä¶"` with `\" \\ \n \t` escapes (unterminated/unknown escape ‚Üí lex error). A lone `.` is member access; `.` before a digit is still a number (`.5`) ‚Äî that branch runs after the two-char pass (`..` is a range) and before the simple table. **Don't add `[`/`{` to the implicit-multiplication cases in `term()`** ‚Äî `a[0]` must be indexing, never `a * [0]`.
- Postfix indexing/member access live in `Parser.postfix()`, between `power()` and `primary()` (binds tighter than `^`, chains: `people[0].age`, `grid[1][2]`).
- **Compact map keys**: `{b:1}` lexes its `b:1` as a cell-reference token (letters + `:` + digits, no space). `Parser.mapLiteral` decomposes a cell-reference token in key position back into key + number value. To support this, the cell-reference TOKEN preserves column case as typed (resolution via `CellAddress.columnIndex` is case-insensitive regardless). A bare `:` outside a map literal is a parse error (no longer a lex error).
- Cells reject structure results: `Spreadsheet.display(of:)` maps `.number`‚Üívalue, `.string`‚Üítext (string-valued formulas are labels ‚Äî `="Q" + quarter` ‚Äî and behave like text when referenced: skipped in ranges, error on direct numeric use), `.array`/`.map`/`.record`‚Üíerror. `expectsOperand` (point mode) includes `[ { :`.

### Data types (`data` declarations)

`data Person { name: String, age: Number, active: Boolean } # doc` declares a record type. The full language behavior is specified in `Features/datatypes.feature` + `docs/ANZAN.md ¬ß7`; the implementation invariants:

- **Parsing**: `Parser.dataDefinition()` ‚Äî `data` is a CONTEXTUAL keyword, committed only by the exact token shape `data Ident {` (so `data = 5` stays an assignment; `data` is NOT in ReservedNames). Type names must start uppercase; scalar field types (Number/String/Boolean) parse case-insensitively into `DataFieldType`, and an uppercase TYPENAME parses as `DataFieldType.record(name)` ‚Äî **records nest** (`data Line { a: Point, b: Point }`), checked at construction (the value must be a record of that type; existence is NOT required at declaration, so forward refs work and persistence restores types-then-variables). No list-typed fields yet. Nested validation is O(1)/field (records are immutable + already-validated ‚Äî no recursive re-checking, no cycle risk).
- **Named arguments are sugar for one map**: `argumentList()` detects `ident :` (or a compact MULTI-letter cell-reference token like `age:36` ‚Äî real columns are single letters) in first position and parses `namedArguments()` into a single `.mapLiteral` ‚Äî so `Person(name: "Ada")` ‚â° `Person({name: "Ada"})` for free, for EVERY call (an ordinary function called with named args just receives a map). Compact single-letter `f(a:1)` stays a cell reference (documented; spaced `f(a: 1)` works). There is deliberately NO positional constructor (user decision).
- **Construction**: `Evaluator.construct` validates all fields present / typed / nothing extra and canonicalizes to declaration order; Boolean fields are strict (exactly 0/1). Resolution order adds data types after env functions, before function-valued variables ‚Äî in `call(name:)`, `tailStep`, AND `apply(function:)`'s `.user` case (constructors travel by name: `map(Person, maps)`); keep all three in sync. Cross-collisions with builtins and user functions are rejected both ways; redeclaring your own type is allowed.
- **`Value.record(RecordValue)`** carries typeName + entries + `booleanFields` (NO back-reference to DataType ‚Äî instances outlive redefinitions). Records read like maps (`mapValue(forKey:)`, member/subscript, keys/values/len) but never equal a plain map; description is the constructor-call form with Boolean fields as true/false (re-parses while the type exists).
- **`toJson(value, option?)`** (DataFunctions) uses `Value.jsonText` ‚Äî pure string building, full-precision numbers, JSON escaping incl. control chars; Boolean record fields emit true/false. Functions refuse. **Pretty is the DEFAULT** (user decision); the option is `Json.Pretty`/`Json.Compact` (user decision: named constants over a magic boolean) ‚Äî `Json` is a reserved constant MAP in the Environment subscript (`Constants.json`) whose members are the plain strings "pretty"/"compact", so no new value machinery; `man()` covers constants via the `constants` lookup in `documentation(for:)`. Display: `EvalOutcome.rawBlock` (engine) surfaces MULTI-line string results raw, and both hosts honor it ‚Äî the log renders them via `HistoryEntry.Outcome.info` (plain block, like man()), the CLI prints them unquoted ‚Äî while single-line strings keep canonical quoting; don't render pretty JSON as one `\n`-escaped line again. **`fromJson(text)`** is the inverse via `JSONParser` (Eval/JSON.swift) ‚Äî HAND-ROLLED deliberately: Foundation's JSONSerialization round-trips numbers through Double, so JSON number literals must go straight to `BigDecimal(string:)` (sign split off, like the lexer). Refuses `null` (Anzan has no null) and duplicate object keys; handles `\uXXXX` incl. surrogate pairs; 256-level depth cap. Re-typing is composition, not a feature: `Person(fromJson(t))`.
- **Persistence**: `DataType.source` keeps the declaration line (trailing `# comment` = documentation, same contract as UserFunction ‚Äî no separate doc storage). Workbook field `dataTypes` (name ‚Üí source line). Restore ORDER is load-bearing: `Calculator.restoreSession(from:)` does types ‚Üí functions ‚Üí variables, because record variables persist as constructor CALLS that `Value(parsing:)` can't fold ‚Äî `restoreVariables` folds literals fast-path and EVALUATES the rest via `evaluateFormula` (never touches `ans`). The app (`SheetModel.apply`) and the gherkin round-trip step both go through `restoreSession`; don't re-inline the old replace-and-evaluate dance.
- **ùë´ cells**: a plain `data ‚Ä¶ { ‚Ä¶ }` cell classifies as `Cell.Definition.Kind.dataType` ‚Äî sheet-scoped like Œª/ùëñ (same index, duplicate rules, log immutability via `scopedDefinitionOwner`), rendered `CellDisplay.definition("ùë´ Name")`, resolved via `Calculator.scopedDataTypeResolver` ‚Üí `Spreadsheet.definedDataType(named:)`. Builtin-name protection in the cell covers `.function` AND `.dataType` kinds (they share the call namespace; `.variable` doesn't).

### Workbook + History reflection, mutation

A formula can inspect the workbook's own structure ‚Äî `Workbook.worksheets[0].cell("A", 2).value` (object graph) or the flat `cell(col, row)` / `cell(sheet, col, row)` / `sheetName()` / `sheetNames()` / `rowCount()` / `columnCount()` accessors (all READ-ONLY) ‚Äî the calculation log via `History` (log-only; see the History bullet below) ‚Äî and a small set of LOG-ONLY commands changes the workbook (`updateCell`/`addWorksheet`/`renameWorksheet`/`deleteWorksheet`). Full behavior is in `docs/ANZAN.md ¬ß10` + `Features/reflection.feature` (Workbook reads) + session gherkin (mutation undo) + `HistoryReflectionTests` (History); the implementation invariants:

- **The generic seam**: `Value.host(any HostObject)` (Anzan/Eval/Value.swift) is a host-implemented escape hatch ‚Äî the protocol is `member(_:) -> Value?` / `index(_:) -> Value?` / `call(_:_:) throws -> Value` + `typeName`/`description`/`isEqual`. Anzan navigates a `.host` uniformly (`.member`, `[‚Ä¶]`, `.methodCall`) and knows NOTHING of what it wraps ‚Äî keep it that way (no Sheet/Workbook import in Anzan). The handle implementations live HOST-side in `SorobanEngine/Sheet/WorkbookReflection.swift` (`WorkbookObject`/`WorksheetCollection`/`WorksheetObject`/`CellObject`).
- **Method-call syntax**: `recv.name(args)` is `Expression.methodCall` ‚Äî parsed in `Parser.postfix()` (the `.dot` case branches to a method call when a `(` follows, else member access), evaluated by routing to `object.call(name, args)`; it requires a `.host` base. Distinct from a free `name(args)` call and from member access.
- **Two new Calculator resolvers** (wired by SheetStore, nil in the CLI): `hostValueResolver` (a bare name ‚Üí a host value ‚Äî `Workbook`/`History` ‚Äî `(name, inLog) -> Value?`, checked in the evaluator's `.variable` case AFTER user variables, before the function-value fallbacks; the `inLog` flag lets a name be log-only ‚Äî `History` returns nil in a cell, where it then degrades to a TEXT LABEL via the unknownVariable‚Üítext rule, NOT an error) and `hostFunctionResolver` (a free call ‚Üí a reflection function, checked LAST in `call(name:)` so a user's own `cell(x) = ‚Ä¶` shadows it).
- **Sendable + live reads**: handles hold the store/sheet/grid **weakly** (a stored `w = Workbook.worksheets[0]` never keeps a removed sheet alive; reads after teardown throw cleanly) and are `@unchecked Sendable` on the engine's single-threaded discipline. Cell reads route through the ordinary `numericValue`/`displayValue` path, so **dependency edges and cycle detection come for free** ‚Äî `=cell("A",1).value+1` recalcs when `A:1` changes; unqualified `cell("A",1)` follows the owning sheet via `scopeSheetItem`. Don't snapshot instead (a frozen copy wouldn't record edges).
- **Mutation (third resolver, log-only)**: `hostMutationResolver: (name, args, inLog) -> Value?` ‚Äî checked LAST in `call(name:)` (after `hostFunctionResolver`), so a user's `updateCell(‚Ä¶)` shadows it. The `inLog` flag is `Calculator.run`'s `allowingMutation`, true ONLY on the `evaluate` (log) path and false on `evaluateFormula` (cell recalc) ‚Äî a mutation from a cell throws "runs in the calculation log, not a cell" (recalc must stay reproducible ‚Äî the rand() principle). The flag rides the Evaluator instance, so it threads through nested user-function applies (a log call to `f()` whose body mutates is allowed; the same `f()` in a cell is not). `SheetStore.installMutation` wires the DIRECT default (mutates the store, no undo ‚Äî what the CLI/engine tests see); the app's `SheetModel.installMutationOverride` REPLACES it post-init to route the same commands through `applyEdit`/`renameActiveSheet`/`removeActiveSheet` (undoable, persisted, UI-refreshing). `SheetStore.mutationNames` is the shared gated set; public seams `cellTarget(of:)` / `sheetIndex(forTarget:)` / `worksheetHandle(at:)` / `rawText(from:)` keep host-object internals private from the app override. `updateCell` writes raw from a value (number‚Üídigits, string verbatim, `""` clears); `renameWorksheet` rewrites references via `ReferenceRewriter.renamingSheet` like the UI rename.
- **History reflection (the calculation log, read-only + log-only)**: `History` resolves to an ARRAY of entry handles (NOT a host object ‚Äî so `len`/`[i]`/`map`/`filter`/`first`/`last` work natively; arrays are 0-based, no `[-1]`, use `last(History)`; `e` is reserved (Euler), so HOF params are `entry`). Handles live in `SorobanEngine/Sheet/HistoryReflection.swift`: `HistoryEntryObject` (member `.input`/`.value`/`.text`/`.kind`/`.isError`/`.referencesCells`/`.note`) over a host-neutral `LogRecord` fed by the `LogSource` protocol (the clean host API ‚Äî engine-tested with a stub). The log itself is a UI-FREE model: **`LogStore`** (`App/Sources/Session/LogStore.swift`) owns the `[HistoryEntry]` tape + persistence (`log.json`) and conforms to `LogSource` DIRECTLY ‚Äî so there's no adapter and no `MainActor.assumeIsolated` (it's a plain `@unchecked Sendable` class, not `@MainActor`). This mirrors `SheetStore` for the grid: `CalculatorSession` is a thin view-model over it (`entries` proxies `log.entries`; observation bridges through `logGeneration`, the grid's `generation` pattern, since `LogStore` isn't `@Observable`; `LogStore.persists=false` in tests, like `autosaveToScratch`). `SheetStore.logSource` (set to the `LogStore`) is the seam; `installReflection`'s resolver returns the array only when `inLog` (in a cell ‚Üí nil ‚Üí text label). `kind` ("value"/"error"/"comment"/**"info"**/"function"/"datatype") is DERIVED from the outcome flags (error/comment/info) + parsing the stored input for function/datatype (a function def is logged as a value of its signature, so the outcome alone can't classify it); `referencesCells` is the input AST's `containsCellReference`; `value` re-parses the result text (`Value(parsing:)`, lossless for numbers/strings); the `HistoryEntry.Outcome ‚Üí LogRecord` map is `LogRecord(_:)` in LogStore.swift. **"info" = display-only**: `man()`/pretty-JSON AND any result carrying a host handle (a bare `Workbook`, a `History` dump) ‚Äî `submit` routes those to `.info` via `Value.containsHost` because their `Workbook(‚Ä¶)`/`[LogEntry(‚Ä¶)]` rendering isn't re-parseable (so they're shown but not recallable, `.value` nil; same reason cells reject host results). Bare `History` works but dumps opaque handles ‚Äî `len(History)` is the size. NO persistence change ‚Äî everything derives from the input + outcome the log already stores. Reflects the TAPE (what you did); the live grid is Workbook reflection, the current names are the environment. Tested both ways: `HistoryReflectionTests` (engine, stub `LogSource`) AND `session.feature` (the session Gherkin builds a real `LogStore` and queries `History` live ‚Äî `LogStore`+`HistoryEntry` are in the `SorobanSessionTests` target). Deferred: mutation (clearHistory/delete), `replay()`, CLI `History`, environment reflection.

### Worksheets

- `SheetStore` (engine) owns ordered named `Sheet`s (grid + per-sheet layout), the 256 cap, and name validation (‚â§128 chars, unique case-insensitive, no `!`/`'` ‚Äî they'd break the reference syntax). **Owning-sheet invariant**: an unqualified `A:1` inside a formula resolves against the sheet that OWNS the formula, via the shared `ResolutionContext`'s current-sheet stack (`Spreadsheet.displayValue` pushes/pops); only log input follows the active tab. Cycle detection is context-wide, keyed by (sheet identity, address) ‚Äî never move it back per-sheet, cross-sheet loops would hang.
- References are **by name**, and renaming a sheet AUTO-REWRITES every `Old!A:1` / `'Old'!A:1` qualifier across all sheets (`ReferenceRewriter.renamingSheet` via `SheetModel.renameActiveSheet` ‚Äî token-precise, bare-or-quoted respelling). Undo: `SheetEdit.Kind.renameSheet` is pushed BEFORE the per-sheet rewrite steps (the named-cell ORDER rule); its undo/redo finds the sheet by the names the EDIT carries (the tag can't be trusted across the rename) and retags both stacks. Rename/remove call `store.recalculate()`.
- Sheet-qualified syntax: `Budget!A:1` / `'Q1 Budget'!A:1` (`.bang` + `.quotedName` tokens; `!=` is matched first by the two-char pass; a quoted name NOT followed by `!` is a named-cell reference ‚Äî see Named cells).
- App: `SheetModel` proxies the active sheet; undo entries are sheet-name-tagged (undo jumps to that sheet; removed sheets drop their entries; renames retag). Tab UI is `SheetTabBar` ‚Äî only the active tab is visible (user design), menu for switching, double-click rename via simultaneousGesture.
- Point-mode reference insertion is same-sheet only in v1 (switching tabs commits the editor).
- **Data sheets**: `Sheet.data: DataSheet?` marks a sheet backed by a `DataStore` (SQLite, engine-tested) table; resolvers route by kind, and data rows are bounded by the TABLE, not `Spreadsheet.rowCount`. The working DB lives at `working-data.sqlite` (App Support); `WorkbookManager.open` copies the package's `data.sqlite` over it (`prepareWorkingDatabase`), saves copy it back via `WorkbookPackage.write(databaseURL:)`. `data.sqlite`'s presence in a package is meaningful: it exists iff data sheets exist. Imports go through `SheetModel.importCSV` (names sanitized/uniquified, columns capped at 26); removal drops the table. **Import is a copy** ‚Äî data sheets are editable, and `SheetModel.write(_:at:on:)` routes edits to `DataSheet.setRawValue` (bounds-checked against the table's fixed shape; the source CSV is never touched). Data-sheet edits skip the scratch journal in `persistCellEdits` (SQLite is the durability; they can't replay through the grid journal) and trigger `store.recalculate()` instead, because data cells aren't dependency-graph nodes. Data sheets never own formulas and render at most 10,000 rows (`visibleRowCount`; selection clamps to `visibleRowCount`/`visibleColumnCount`, not the grid bounds).
- `.soroban` is a document **package** (`WorkbookPackage`): directory with `workbook.json` (+ `data.sqlite`); legacy flat JSON reads transparently; writes are atomic temp-dir swaps that also upgrade flat files in place. The exported UTI + CFBundleDocumentTypes live in project.yml's `info:` block (regenerate after edits); Finder open arrives via `.onOpenURL`.
- **CSV, three doors** (don't conflate): `WorkbookManager.openCSV` (‚áß‚åòO) = NEW workbook ‚Äî grid cells when the file fits 1000√ó26 (auto-detection makes numbers numbers and text labels), else falls back to a data sheet; `importData` = data sheet into the CURRENT workbook; `exportCSV` = the active sheet's computed VALUES via `SheetModel.activeSheetCSV` (numbers plain/unformatted, controls export their value, Œª/ùëñ cells export their raw source, errors as #ERR; used bounds for grids, the full table for data sheets). `CSV.encode` (engine, tested) is `parse`'s exact inverse ‚Äî quote only when needed, `parse(encode(rows)) == rows`. Save panels set `isExtensionHidden = false`.

### Engine conventions

- Errors are always `EngineError`; lex/parse errors carry character offsets that the UI renders as a caret under the offending column. Preserve positions when changing Lexer/Parser.
- Finance functions follow the spreadsheet sign convention (outflows negative); `rate`/`irr`/`solve` solve numerically (Newton + bisection fallback) in the Double domain. Paired-series functions (correl/slope/intercept/forecast/sumproduct, like xnpv/xirr before them) SPLIT their flattened arguments evenly ‚Äî two equal ranges read naturally; `percentile` takes p LAST for the same reason. Golden values in `FinanceTests` are spreadsheet-checked ‚Äî don't "fix" them to match changed code without re-deriving.
- Swift 6 strict concurrency is on: anything stored in a global (function lists, registry) must be `Sendable`; function closures are `@Sendable`.
- The parser supports implicit multiplication (`2x`, `2(3+4)`, `2 A:1`, `2pi`) via the `term()` loop ‚Äî a value against a NAME/PAREN/CELL. A bare NUMBER as the right operand (`3 4`, or `3 % 4` = `3%` then `4`) is NOT implicit √ó: `term()` throws "a number can't directly follow another value" (it's a missing operator; modulo is `mod()`). Additions to the token set must not break the name/paren/cell cases (a new token kind that can start an operand needs a case there).

### Cell formatting

`CellFormat` (engine, Sheet/CellFormat.swift): style flags + alignment + semantic `PaletteColor`s + `NumberFormat` (general/number/currency/percent/date/hex/binary ‚Äî the last two are the honest "programmer mode": display-only radix, NEVER a semantics switch; a mode-flipped `^`-as-XOR was considered and refused because stored formulas must mean the same thing under every UI state, the rand() principle at the syntax level; BOTH hosts' hex echo rides `Calculator.usesProgrammerNotation` + `BigDecimal.hexText` ‚Äî the CLI appends to the printed line, the app log carries it as `HistoryEntry.annotation`, display-only and EXCLUDED from Insert/Copy so recallValue stays re-parseable; computed from the RESULT, never `ans` ‚Äî definition lines mentioning 0x must not annotate stale numbers). Invariants: (1) **rendering is pure string/BigInt math** ‚Äî `NumberFormat.rendered` builds digit strings from significand/exponent, never Double or Foundation NumberFormatter, so 40-digit values group exactly; percent is an exponent shift; date uses `CivilDate`. (2) **Display-only** ‚Äî formats live in `Sheet.formats: [CellAddress: CellFormat]` (sparse; `isDefault` entries pruned), never touch `Cell`/the dependency graph/recalc, and empty cells may be formatted. (3) Colors are stored as semantic NAMES; the app maps them to system colors (PaletteColor.color in CellView.swift) so they adapt to light/dark ‚Äî don't switch to absolute RGB. (4) Codable is custom-compact (non-default fields only; unknown future `style` strings degrade to general); workbook field is `sheets[].formats` keyed "A:1", default-decoded. (5) App: ALL format mutations route through `SheetModel.applyFormat` (undoable ‚Äî `SheetEdit.Kind.formats`, parallel to `.cells`; format edits persist via the snapshot path, NOT the cell journal); toggles use the all-set‚Üíclear rule (`toggleStyle`). (6) `CellView` gets `format` as an Equatable let (in `==`) ‚Äî the render invariant holds; `FormatActions` is the single shared definition for the menu-bar Format menu (shortcuts ‚åòB/I/U, ‚áß‚åòX, ‚åÉ‚åò./,) AND the per-cell context menu ‚Äî where it sits inside a "Format" SUBMENU under Cut/Copy/Paste/Delete (user feedback: an all-formatting context menu read poorly; editing verbs come first). Context-menu clipboard verbs use `SheetModel.copySelectionToPasteboard`/`cut‚Ä¶`/`pasteFromPasteboard` (direct NSPasteboard ‚Äî the ‚åòC/‚åòX/‚åòV routes stay on the responder-chain `onCopyCommand` handlers); every action retargets via `retargetSelection(toInclude:)` when the clicked cell is outside the selection. Errors/definitions keep fixed styling (fill still applies).

### Named cells ('Projected Rate')

A name for a cell LOCATION (vs ùëñ definitions, which name VALUES). Syntax reuses the quoted-name token: a bare `'Name'` (previously a parse error) is `.nameReference(sheet: nil, ‚Ä¶)`; `Sheet!'Name'` parses via `qualifiedReference` (which now accepts a quotedName after `!`). Resolution: `Calculator.nameResolver` ‚Üí SheetStore routes like cells (owning sheet via context, active from the log; data sheets have none) ‚Üí `Spreadsheet.numericValue(forName:)` ‚Üí the ordinary cell-read path, so dependency edges and cycle detection are free. Storage: `Spreadsheet.cellNames` ([CellAddress: String]; ‚â§64 chars, unique per sheet case-insensitive, no `'`/`!` ‚Äî `setCellName` validates and `invalidateEverything`s); persisted as `sheets[].names` keyed "A:1". **Rename auto-rewrites references; delete asks break/inline/cancel** ‚Äî both via `NamedCells.rewriting` (token-precise; scoping mirrors resolution: unqualified matches only on the owning sheet, qualified matches the owning sheet's qualifier anywhere; a quoted SHEET qualifier with the same spelling ‚Äî `'Rate'!A:1` ‚Äî is skipped). Cross-sheet rewrites land as one undo step PER affected sheet (the undo model is sheet-tagged). Point mode inserts `'Name'` for named cells (range extension falls back to addresses via `lastInsertedAddress`). Name operations are undoable (`SheetEdit.Kind.name`); **push ORDER is load-bearing**: rename pushes the name step BEFORE its rewrite steps, removal pushes it AFTER ‚Äî so the ‚åòZ sequence always ends (or, for removal, begins) with name and formulas agreeing. The name-step undo uses `try? setCellName` (a later edit may have claimed the old name ‚Äî skip, don't crash).

### Control expressions (slider / stepper / checkbox / dropdown)

A cell whose expression is a LITERAL-argument control call renders as an interactive control: `slider(value, min, max[, step])` (drag), `stepper(value, min, max[, step=1])` (‚àí/+ clicks), `checkbox(state)` (toggle, true/false literals), `dropdown(value, [options])` (menu; **the cell's value IS the selected option** ‚Äî string or number literals). Named when it's a ùëñ definition (`rate = slider(‚Ä¶)`, all definition semantics inherited), anonymous as `=slider(‚Ä¶)`/plain. Detection is unified in `Control.display(for:)` (Sheet/Controls.swift ‚Äî the SliderInfo/CheckboxInfo/DropdownInfo home) ‚Üí `CellDisplay.slider/.stepper/.checkbox/.dropdown`. Invariants: (1) the value argument IS the storage ‚Äî non-literal args mean it's not a control (falls back to normal evaluation/ùëñ rendering); the builtins stay pure (clamp / 1-0 / pass-through; registry-documented, work headlessly). (2) Interaction rewrites the storage literal via `Control.rewriting(_:toLiteral:)` (token-precise splice ‚Äî preserves spacing/# comments; handles signed numbers, true/false, "strings") through `applyEdit` ‚Äî one undo step, one journal entry. Sliders alone add **drag = preview, release = commit** (column-resize discipline): `Spreadsheet.setSliderOverride`/`clearSliderOverride` feed display AND `definedValue` mid-drag with TARGETED invalidation ‚Äî only the cell + its recorded readers, never `store.recalculate()`, which made drags O(workbook) per tick ‚Äî gated on quantized-value change only ‚Äî the old 30Hz floor held the knob behind the cursor and is gone (`SheetModel.previewSlider`); the others commit immediately (`commitControl`). (3) Detection runs AFTER the definition duplicate/builtin checks (a shadowed control must show its error, not a working widget). (4) Control cells read as their value in references and ranges (checkbox ‚Üí 1/0; string dropdowns act like text: error direct, skipped in ranges). (5) The cell control views are stateless (per-cell @State ban) ‚Äî state comes from the model; gestures defer to point mode when an editor is open. (6) The slider's value label reserves its WIDEST possible text (hidden width template; CellView.widestSliderText) so the track never resizes mid-drag ‚Äî GeometryReader width arrives a layout pass late, which made the knob land off-cursor for a frame, then correct.

### Sheet-scoped definitions (Œª / ùëñ / ùë´ cells)

A PLAIN `tax(x) = ‚Ä¶` / `rate = expr` / `data Pt { ‚Ä¶ }` cell (no `=` marker) classifies as `Cell.Content.definition` ‚Äî the third sibling of formula/text. Rendering is `CellDisplay.definition("Œª tax(x)" / "ùëñ rate" / "ùë´ Pt")` (accent + italic in CellView; the value never shows ‚Äî user design). Key invariants:

- **Index**: `Spreadsheet.definitions` ([lowercased name ‚Üí SheetDefinition]) is rebuilt by full scan on any definition edit and on `load` ‚Äî one case-insensitive namespace per sheet; earliest address wins, later duplicates display errors; function defs colliding with builtins error in the cell. `setCell` touching a definition calls `invalidateEverything` (Œª/ùë´ calls leave no graph edges ‚Äî same hammer as log-variable changes) with ONE carve-out: a ùëñ cell redefining the SAME variable name (`isSameVariableRedefinition` ‚Äî every control commit lands here) invalidates targeted, sound only because `definedValue` records a definition-cell‚Üíconsumer edge per read; don't widen the carve-out to Œª/ùë´ or renames.
- **Resolution**: Evaluator gets `resolveScopedFunction`/`resolveScopedVariable`/`resolveScopedDataType` (wired by SheetStore via `scopeSheet` = `context.currentSheet ?? activeSheet.grid` ‚Äî owning sheet mid-evaluation, active tab from the log). Precedence: locals ‚Üí scoped ‚Üí environment for variables; registry ‚Üí scoped ‚Üí log functions ‚Üí data types ‚Üí function-valued variables for calls. Sheet definitions SHADOW log globals everywhere, including the log.
- **Laziness + dependencies**: ùëñ expressions evaluate per lookup via `evaluateFormula` ‚Äî cell reads inside them attribute to the CONSUMING formula (the context's currentKey), and the lookup itself records the definition CELL as read ‚Äî so slider previews and same-name redefinitions invalidate only actual readers. Self-reference is cut by `resolvingDefinitions` ("circular definition"); cell‚Üîdefinition loops fall out of the ordinary resolving-set check.
- **Immutability**: `Calculator.evaluate` intercepts log assignments/definitions whose name has a `scopedDefinitionOwner` ‚Üí "defined in cell Budget!A:3". `evaluateFormula` rejects `.assignment` (only reachable via `=name = value`; plain forms classify first) ‚Äî cells NEVER write globals anymore.
- Referenced numerically, definition cells act like text (error direct, skipped in ranges). Persistence is free (definitions are cell raws); `man()` finds Œª cells via `scopedFunctionResolver` in `documentation(for:)`; bare Œª-cell names become function values as `.lambda` (structural ‚Äî a name can't re-resolve them).

### Spreadsheet / cell references

- `A:1` is a token (`.cellReference`), lexed only when a letters-only identifier is followed by `:digits` ‚Äî or with `$` pins (`$A:$1`): pinColumn/pinRow ride the TOKEN only (copy-time data for fill/paste; the AST and evaluation never see them, `$A:$1` ‚â° `A:1`; a dangling `$` is a loud lex error). Token-precise reference rewriting (structural shifts, relative fill/paste adjustment, sheet renames) lives in `ReferenceRewriter` (SorobanEngine/Sheet) ‚Äî it skips compact map keys (`{b:1}`) and multi-letter named-arg columns, pairs range corners only across a real `..` token, and splices `refError()` (an always-throwing core builtin) over references killed by a delete or pushed off the grid. It flows through the evaluator via `Calculator.cellResolver`, which the app wires to `Spreadsheet.numericValue` ‚Äî that's how cell refs work from the log too.
- Cell classification is split in two, deliberately. **Static** facts live in `Cell.init` (Cell.swift), computed once at commit: `=‚Ä¶` forces formula (parse errors included ‚Üí `#ERR`); `"‚Ä¶"` forces text, quotes stripped; unparseable ‚Üí plain text; anything else parses into a stored AST as a `.candidate`. **Dynamic** facts live in `Spreadsheet.evaluate(_:)`, per recalc: candidates that evaluate are values; on failure the **error kind** decides ‚Äî unknownVariable/unknownFunction ‚Üí text (protects labels like "Q1 revenue", which parses as `Q1 * revenue`; note this means a label can *become* a value when the log later defines the variable), any other failure (division by zero, domain, arity) or any cell-referencing expression ‚Üí `#ERR`. Don't move candidate resolution into `Cell.init` ‚Äî it depends on the live environment. Don't change these rules casually ‚Äî they're load-bearing UX.
- Cells are parsed **once** (at commit, in `Cell.init`); recalc evaluates stored ASTs via `Calculator.evaluateFormula(_ expression:)`. Don't reintroduce string re-parsing into the recalc path.
- Cell evaluation goes through `Calculator.evaluateFormula` (never updates `ans`); `evaluate` (log path) does. Keep that asymmetry.
- `EvaluationEnvironment` is a **class** because formula evaluation re-enters the calculator through the resolver (cell ‚Üí resolver ‚Üí another formula); threading it `inout` caused a Swift exclusivity crash. Don't convert it back to a struct.
- It's named `EvaluationEnvironment` (not `Environment`) to avoid colliding with SwiftUI's `@Environment` in app files importing both modules.
- Grid interaction: `SheetModel.selected` is the selection **anchor**; `selectionExtent` (shift-click / shift-arrows) makes it a rectangle (`selectionRect` normalizes). `editing` (double click / Return) is separate. Keyboard commands hang off the grid container's focus (`.onCopyCommand` etc. in SpreadsheetView; arrows use `onKeyPress(keys:)` to read the shift modifier ‚Äî `onMoveCommand` can't). Copy/cut produce **TSV of raw contents** (interops with Excel/Numbers); a COPY also writes a custom pasteboard type (`SheetModel.cellsPasteboardType` ‚Äî anchor + TSV) so an in-app paste ADJUSTS relative references by the move offset (`ReferenceRewriter.adjustingRelative`; `$` pins hold; off-grid refs become `refError()`; the TSV must equal the plain-string content or the custom data is stale and paste is verbatim); CUT stays plain TSV (cut-paste keeps refs verbatim ‚Äî Excel's no-adjust-on-move). External pastes are verbatim. Paste fills from the anchor, clipped, as ONE undoable edit. **Fill Down/Right (‚åòD/‚åòR, Edit menu)**: top row/left column of the selection is the source, each target adjusts by its offset; a single-cell selection fills from the neighbor above/left. SpreadsheetView's onCopy/onCut hand the writes to the model (returning []) so both pasteboard types land.
- **Point mode** (Excel-style): while a cell editor is open and its draft "expects an operand" (`Calculator.expectsOperand` ‚Äî engine-tested), clicking another cell inserts its reference instead of committing; re-click replaces the inserted reference, shift-click extends it to a range. Mechanics: the draft lives in `SheetModel.editingDraft` (NOT editor-local @State); focus-loss commits wait a 250ms grace window (`editorLostFocus`) because mouseDown steals focus before the tap arrives ‚Äî one async hop is NOT enough, it fires between mouseDown and mouseUp; cell clicks resolve the edit explicitly (insert cancels, plain clicks `commitOpenEditor`), regained focus cancels, the timeout covers clicks outside the grid ‚Äî `insertReference` cancels the pending commit and bumps `editorRefocusTrigger` to take focus back. All cell clicks route through `SheetModel.handleCellClick`.
- **Structural edits** (insert/delete rows & columns): `SheetStore+Structure.swift` owns the op ‚Äî raw rewrites across all grid sheets via `ReferenceRewriter.shifting` (recorded at PRE-move addresses), then a content move on the edited sheet (cells/names/formats/sizes re-key; deletes capture the removed slice) committed through `grid.load()`. `StructuralChange` is the undo record: undo = `store.revert` (exact inverse ‚Äî move back, restore removed slice, re-apply OLD raws), redo = re-execute (deterministic ‚Äî post-undo state equals pre-op state). Insert REFUSES when occupied cells/names would fall off the grid; data sheets refuse entirely. App: `SheetModel+Structure.swift` (`insertRows`/`deleteRows`/`insertColumns`/`deleteColumns` ‚Üí `SheetEdit.Kind.structure`); persistence is the SNAPSHOT path, never the cell journal (replay can't represent shifts). UI: context menus on row/column headers (RowHeaderMenu/ColumnHeaderMenu in SpreadsheetView ‚Äî pluralized over the selection when the clicked header is inside it).
- Undo/redo lives in `SheetModel` (`applyEdit` pushes grouped `[CellChange]` steps, capped at 100; stacks clear on open/new): grid content only ‚Äî the log is history, not document state. Route any new mutation path through `applyEdit`, never `spreadsheet.setCell` directly, or it won't be undoable.
- **Grid render performance invariants**: (1) `CellView` must NOT read the observable model ‚Äî `GridRowView` reads selection/display once per row and passes everything down as Equatable lets; `CellView` conforms to `Equatable` (nonisolated `==`) so SwiftUI skips unchanged cell bodies. Cells observing `sheet.selected` directly once made every click invalidate ~1,000 visible bodies. (2) Per-cell `@State`/`@FocusState` is banned ‚Äî editor machinery lives in `CellEditorView`, instantiated only for the single editing cell. (3) Editor focus is grabbed via `DispatchQueue.main.async` in `onAppear` (synchronous FocusState writes don't stick mid-update inside the lazy grid). (4) Perceived speed: profile/judge with a **Release** build ‚Äî Debug SwiftUI with thousands of views is 5‚Äì10√ó slower. (5) The cell double-tap is a `simultaneousGesture`, NOT a stacked `onTapGesture(count: 2)` ‚Äî stacking makes SwiftUI hold every single click for the double-click window (~0.3s of pure input latency); `SheetModel.select` is deliberately order-tolerant for the second click's dual delivery. (6) Column/row resizing is preview-based: drags update only `SheetModel.columnResizePreview`/`rowResizePreview` (observed solely by the guide-line overlays in SpreadsheetView); the real `columnWidths`/`rowHeights` mutate once on release ‚Äî applying live would re-run every alive row body per drag tick.
- All `"A:1"` key / column-name / 0-vs-1-based row conversions live on `CellAddress` (`init?(key:)`, `init?(columnName:rowNumber:)`, `columnIndex(forName:)`, `columnName(forIndex:)`) ‚Äî don't re-implement them elsewhere.
- `SheetModel.generation` is the observation bridge: the engine `Spreadsheet` isn't @Observable, so cell views re-render by reading `generation`, bumped on every commit/recalc. Anything that can change cell values (e.g. a log submission assigning a variable) must call `sheet.recalculate()`.
- Sheet contents persist as `["A:1": raw]` JSON at `Application Support/Soroban/sheet.json` (sandbox container) ‚Äî but **only while untitled**; see workbooks.

### Workbooks (Save/Open)

- `.soroban` files are the engine's `Workbook` codec (Persistence/Workbook.swift): versioned JSON envelope of `cells` + `variables` + `functions` + `dataTypes` + `columnWidths`/`rowHeights` (newer fields decode with defaults for older files). The schema is specified in `docs/FORMAT.md` ‚Äî keep it in sync when evolving the format. Bump `currentVersion` and keep decode backward-compatible; decode rejects future versions. Open goes through `Calculator.restoreSession(from:)` (types ‚Üí functions ‚Üí variables ‚Äî order is load-bearing, see Data types).
- `WorkbookManager` (App, Session/) owns `fileURL`/`isDirty` and the NSOpen/NSSavePanels. Autosave rule: untitled ‚Üí `SheetModel.autosaveToScratch` keeps writing the scratch file (a full `Workbook`: cells + variables + functions + data types + layout); named file ‚Üí changes only mark dirty, ‚åòS writes. Variable/function/data-type changes dirty the workbook too ‚Äî detected via `EvaluationEnvironment.changeCount` (compared around `submit()` in CalculatorSession; don't go back to snapshotting dictionaries).
- Quit with a dirty *named* workbook ‚Üí Save/Discard/Cancel via `AppDelegate.applicationShouldTerminate` (SorobanApp.swift). Untitled needs no prompt (autosaved).
- The sandbox entitlements are defined in `project.yml` (`entitlements:` block; `xcodegen generate` writes `swift/App/Soroban.entitlements`). `com.apple.security.files.user-selected.read-write` is required for the panels to actually grant file access.
- Deferred: Finder double-click association (CFBundleDocumentTypes + onOpenURL), recent-files menu.

### Theming

`Theme` is Codable JSON (`#RRGGBB` colors). Ten built-ins live in `swift/App/Resources/Themes/*.json` ‚Äî but XcodeGen copies them **flat** into the bundle root, so ThemeManager scans both the root and a `Themes/` subdir (don't "simplify" that away); user themes load from `~/Library/Application Support/Soroban/Themes/` at launch. `Theme.fallback` in ThemeManager.swift is the compiled-in safety net and must stay in sync with the JSON schema. Font family/size are app-level overrides in ThemeManager (`fontFamilyOverride`/`fontSizeOverride`, UserDefaults-persisted), applied inside the computed `current` ‚Äî views read `current` and get them for free; the Settings picker binds `currentName`, not `current` (an overridden `current` never equals a listed theme). Only fixed-pitch families are offered: error-caret padding and grid alignment require monospace.

### Environment inspector

`InspectorView.swift` is a trailing sidebar (240pt, both views, ‚å•‚åò0 + toolbar toggle, `inspectorVisible` UserDefaults-persisted) listing every live name with value + PROVENANCE. TWO data sources merge per section: the LOG half on `CalculatorSession` (`logVariables`/`logFunctions`/`logDataTypes` over `environment.user*`) and the SHEET half on `SheetModel+Inspector.swift` (`sheetVariables`/`sheetFunctions`/`sheetDataTypes`/`namedCells` ‚Äî scan every non-data sheet's `definitions`/`cellNames`, values via the now-public `definedValue`/`numericValue`). Categorizing needs `Spreadsheet.SheetDefinition.kind`/`.signature` (public ‚Äî the app can't see package-internal `Cell.Definition`). Refresh rides BOTH observation bridges: `session.environmentGeneration` (bumped in `submit` when `changeCount` moved ‚Äî the log half) and `sheet.generation` (the sheet half); InspectorView touches both at the top of `body`. Cell-defined rows click to jump (`session.jumpTo` ‚Äî activate sheet, select, switch to grid); rows are stateless (no per-row @State, the grid-perf rule).

### Autocomplete

Candidate generation is `Calculator.completions(forPrefix:)` AND word extraction (`Calculator.trailingIdentifier(of:)`) in the engine (both tested); the app side (`CalculatorSession.refreshSuggestions` + `SuggestionsView`) only handles state and keys. ‚Üë/‚Üì are overloaded: suggestion navigation while the list is open, input history otherwise ‚Äî preserve that ordering in `InputBarView`. `suppressNextSuggestionRefresh` exists because programmatic input writes (history recall, log clicks, accept) trigger the same `onChange` as typing; without it the list pops open after every recall.

### Log UX decision

Log text (expressions/results/errors) is deliberately selectable plain `Text`; recall/insert/copy actions live in the right-click context menu, not on left-click ‚Äî buttons would swallow text selection. Keep it that way.

**Log persistence**: the visible tape (`CalculatorSession.entries`) is a GLOBAL running log (one tape across the whole session, like the ‚Üë/‚Üì `inputHistory` ‚Äî NOT per-workbook), persisted to `Application Support/Soroban/log.json` (capped at 500, rewritten whole on each submit and on Clear Log, reloaded in `init`). `HistoryEntry` is `Codable` for this (its `id` is omitted from `CodingKeys` and regenerated on load ‚Äî it's only SwiftUI row identity). The empty-state welcome shows only when the tape is empty, sampling 3 of `welcomePool` per launch.

**Environment inspector** & double-click insert: see the Environment inspector section above; inspector rows insert name/value via `SheetModel.insertIntoEditor` (open cell editor) or `session.insert` (log input).
